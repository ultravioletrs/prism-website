Prism AI - Confidential AI Platform for Secure Multi-Party Collaboration

Prism AI is a confidential computing platform designed to help organizations securely execute sensitive AI workloads inside Trusted Execution Environments (TEEs). It enables the deployment of encrypted virtual machines across private, hybrid, and public cloud infrastructures while preserving data confidentiality.

Prism AI leverages trusted execution environments (TEEs) and confidential cloud infrastructure to enable secure multi-party computation and privacy-preserving AI workflows.

At its core, Prism AI provides a robust Hardware Abstraction Layer (HAL) and a secure runtime environment within each confidential VM, orchestrating and continuously monitoring isolated computations. This architecture forms a strong foundation for secure multi-party computation (SMPC) and collaborative AI workflows.

Prism AI enables privacy-preserving AI collaboration by allowing multiple organizations to jointly analyze and train models on sensitive data without exposing raw inputs. Through encrypted execution and secure orchestration, data remains protected at all times.

With Prism AI, organizations can confidently share data, collaborate on AI projects, and innovate within a fully protected environment. The platform combines confidential computing, secure virtualization, and collaborative AI tooling to unlock new possibilities for regulated and data-sensitive industries.

Prism AI Features

Computation Definition and Management

Prism AI simplifies secure computation definition and management, enabling organizations to define, orchestrate, and execute confidential AI workloads inside Trusted Execution Environments (TEEs). This approach ensures efficient workload orchestration while preserving data confidentiality throughout execution.

Fine-Grained Access Control

Prism AI provides fine-grained access control using attribute-based (ABAC) and role-based access control (RBAC) policies. Only authorized entities can access orchestration metadata, while AI models and data are never uploaded to Prism AI—execution occurs exclusively inside user-attested confidential enclaves.

Secure VM Provisioning and Management

Prism AI enables secure provisioning, management, and monitoring of encrypted virtual machines designed for confidential computing workloads. Each virtual machine is isolated to ensure integrity, confidentiality, and runtime security across cloud and on-premises environments.

Hardware Abstraction Layer (HAL)

Prism AI integrates a robust Hardware Abstraction Layer (HAL) and runtime environment within secure enclaves. This layer ensures the secure execution of confidential workloads, providing isolation, hardware-level security, and efficient orchestration across encrypted virtual machines.

In-Enclave Agent

Prism AI integrates a dedicated In-Enclave Agent, acting as a secure execution scheduler and coordinator within Trusted Execution Environments (TEEs). This agent ensures efficient, confidential AI workload orchestration, enhancing security, monitoring, and seamless task execution inside private enclaves.

End-to-End Encrypted Traffic

Prism AI guarantees end-to-end encrypted traffic within Trusted Execution Environments (TEEs), protecting both confidentiality and integrity of all data transmissions. Collaborate on AI workloads with confidence, knowing that all communication and data exchanges are fully secure and private.

Remote Attestation Mechanism

Prism AI implements a robust remote attestation mechanism to verify the integrity of remote systems, ensuring that workloads execute only within trusted and attested execution environments.

Prism AI Architecture

Prism AI, a leading platform for secure collaborative AI, features an intricately designed architecture optimized for confidential computing and privacy-preserving AI collaboration. Its modular and scalable design provides developers and organizations with a robust foundation to build, deploy, and manage secure AI applications with end-to-end encryption, trusted execution environments (TEEs), and advanced orchestration capabilities.

1. Cloud Services: Prism AI leverages a microservices-based architecture, with each service containerized using Docker and orchestrated by Kubernetes. These cloud services include the Computations service, responsible for managing confidential multi-party computations within secure enclaves; the Users service, for user and organization management; the Authorization service, ensuring robust authentication and policy-based access control; and an intuitive user interface (UI) for seamless orchestration.

2. TEE Manager: The Trusted Execution Environment (TEE) Manager is a critical microservice running on host hardware with TEE capabilities. It dynamically deploys and configures TEEs required for confidential computations. After a computation is completed and the result is retrieved, the Manager securely destroys the TEE and all associated data, maintaining strict confidentiality.

3. In-Enclave Agent: The In-Enclave Agent is a lightweight, open-source component running within each enclave. It coordinates algorithm execution, establishes TLS-encrypted channels for secure uploads of AI algorithms and datasets, and provides remote attestation to ensure the integrity of the execution environment.

Open-source Core with Cocos AI

Core components of Prism AI, including the TEE Manager and In-Enclave Agent, are part of the open-source Cocos AI project. This ensures every computational node, whether cloud-based or on-premises, benefits from robust features and reliable TEE orchestration.

Cocos AI runs on every node with TEE hardware, providing essential functionality for TEE creation, management, and secure computation orchestration. Prism AI leverages Cocos AI to inherit advanced capabilities like confidential VM orchestration and remote attestation.

This open-source approach fosters transparency and community-driven innovation, keeping Prism AI at the forefront of privacy-preserving collaborative AI. Users benefit from established, tested features while leveraging continuous improvements contributed by the community.

How Prism AI Works

Step 1

User creates a Computation, defines characteristics, and participants. A secure VM (TEE) is provisioned and prepared for use.

Step 2

Program and Data Providers upload programs and datasets into the enclave using secure CLI and remote attestation.

Step 3

Programs are executed over combined datasets, coordinated by an in-enclave Agent, following the Computation manifest.

Step 4

Results are sent to the Result Consumer user as the algorithm completes execution inside the enclave.
